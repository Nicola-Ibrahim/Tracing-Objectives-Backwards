# Contributing to Tracing Objectives Backwards

First off, thank you for considering contributing to this project! It's people like you that make the research community so great.

## How Can I Contribute?

### Reporting Bugs
- Check the issues to see if it has already been reported.
- If not, create a new issue. Include as much detail as possible, including steps to reproduce the bug.

### Suggesting Enhancements
- Check the issues to see if it has already been suggested.
- If not, create a new issue. Explain why the enhancement would be useful and how it should work.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. If you've changed APIs, update the documentation.
4. Ensure the test suite passes.
5. Make sure your code follows the project's coding style (Ruff is used for linting).

## Coding Standards
- We follow **Domain-Driven Design (DDD)** and **Clean Architecture**.
- Keep your modules focused and avoid coupling between layers (Domain, Application, Infrastructure).
- Use type hints wherever possible.
- Document new functions and classes.

## Development Setup
Refer to the [Usage & Building](docs/guide/usage.md) guide for details on setting up your environment with `uv`.
