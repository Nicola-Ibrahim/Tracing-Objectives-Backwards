# Global Project Rules

1. **Package Management:** Use `pnpm` for all frontend tasks and `uv` for the Python backend. Never use `npm` or `pip` directly.
2. **No Placeholders:** Never write code with placeholders like `// TODO: implement this`. Always write the complete, functioning code.
3. **Commit Standard:** If asked to write a commit message, format it strictly as `type(scope): description`.
