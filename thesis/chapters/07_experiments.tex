\section{bi-objective case}\label{sec:exp_coco}

\subsection{Multi-objective optimization}
A multi-objective optimisation problem (MOP) is defined by decision variables
$\vect{x}=(x_1,\dots,x_n)\in\calX\subset\R^n$ and an objective vector
$\vect{f}=(f_1,\dots,f_m)\colon \calX\to\calY\subset\R^m$. We consider minimization of all objectives.
A general constrained MOP is written as
\begin{align}
\min_{\vect{x}\in\calX}\; \vect{f}(\vect{x}) &= \big(f_1(\vect{x}),\dots,f_m(\vect{x})\big),\\
\text{s.t.}\; g_j(\vect{x}) &\le 0,\quad j=1,\dots,J,\\
h_k(\vect{x}) &= 0,\quad k=1,\dots,K.
\label{eq:mop}
\end{align}
The feasible region $\calX$ is assumed compact. Objectives may be nonlinear, non-convex, and costly to evaluate, which motivates data-driven surrogates and inverse models in later sections.

Trade-offs are formalized by Pareto dominance. For $\vect{u},\vect{v}\in\R^m$,
$\vect{u}\preceq\vect{v}$ holds if $u_i\le v_i$ for all $i$ and $u_j<v_j$ for at least one index $j$.
A feasible point $\vect{x}^\star$ is Pareto-optimal if no feasible $\vect{x}$ satisfies
$\vect{f}(\vect{x})\preceq\vect{f}(\vect{x}^\star)$. The corresponding image in objective space is the Pareto front.
Figure~\ref{fig:pareto_front} illustrates this geometry for a bi-objective case, showing an example non-dominated
set (Pareto front approximation) together with a dominated region induced by a representative Pareto point.

\subsection{Problem and Data}

We generate bi-objective data from COCO BBOB test functions using the \texttt{cocoex} Python
interface, which provides standardized benchmark suites and deterministic function instances.
\cite{cocoexDocs} We focus on four BBOB functions: $f_5$ (Linear Slope), $f_6$ (Attractive
Sector), $f_{10}$ (Ellipsoid), and $f_{20}$ (Schwefel $x\sin(x)$), which span boundary optima
and strong directionality ($f_5$), pronounced asymmetry ($f_6$), ill-conditioning ($f_{10}$),
and highly multi-modal structure ($f_{20}$). \cite{cocoBBOBOverview}

To obtain a two-objective problem from each selected function, we pair two COCO instances of
the same analytic form:
\[
\vect{f}(\vect{x})=\big(f^{(\mathrm{inst}=a)}(\vect{x}),\; f^{(\mathrm{inst}=b)}(\vect{x})\big),
\]
with $(a,b)$ fixed per function (e.g., $a=1,b=2$). Instances implement deterministic shifts and
transformations while preserving the function family, enabling reproducible variation across
runs. \cite{cocoexDocs,cocoBBOBOverview}

For each function-based bi-objective problem, we approximate a Pareto set using NSGA-II under
a fixed evaluation budget and random seeds. \cite{deb2002nsga2} Unlike earlier drafts where
all evaluated pairs were logged for training, in this chapter we retain only Pareto-relevant
records: the final non-dominated archive returned by NSGA-II. Concretely, for each function
we construct a dataset
\[
\mathcal{D}_{\mathrm{PF}}=\{(\vect{x}_i,\vect{y}_i)\}_{i=1}^{N_{\mathrm{PF}}}, \qquad
\vect{y}_i=\vect{f}(\vect{x}_i),
\]
where each $\vect{y}_i$ is non-dominated within the final archive. The resulting size
$N_{\mathrm{PF}}$ depends on the function and evaluation budget.

This Pareto-only dataset matches the intended use case of inverse Pareto design. In many
engineering settings, the available data are not arbitrary samples from the full design space,
but a set of competitive trade-off solutions produced by an optimizer or prior design process.
A bi-objective Pareto front can be interpreted as a menu of performance trade-offs: for example,
a point $\vect{y}^\star$ could represent a target balance between power consumption and signal
fidelity, or between cost and performance in a component design. The inverse model then serves
as a conditional sampler that maps this chosen trade-off back into decision space, returning
multiple plausible designs $\vect{x}$ that realize comparable outcomes.

We split $\mathcal{D}_{\mathrm{PF}}$ into train/validation/test (80/10/10) using a fixed seed per
function. Decision variables are scaled using the COCO domain bounds, and outcomes are
standardized using training-set statistics. Targets for inverse evaluation are taken from the
held-out Pareto outcomes (and, when needed, small perturbations of these outcomes to represent
user-specified targets near the front). This ensures that conditioning queries remain in a
Pareto-relevant region while still probing generalization beyond the exact training points.

\subsection{Objective-space diagnostics}\label{subsec:objective_diagnostics}
%-------------------------------------------------------------------------------

Objective-space diagnostics evaluate inverse estimators as conditional samplers used in a
propose--check loop. For a requested target outcome $\vect{y}^{\star}$, an inverse model generates a
set of $K$ decision candidates $\{\widehat{\vect{x}}^{(k)}\}_{k=1}^{K}$. Each candidate is then checked
with the forward evaluator $\mathcal{E}$ to obtain outcomes
$\widehat{\vect{y}}^{(k)}=\mathcal{E}(\widehat{\vect{x}}^{(k)})$. Because the output is a set rather
than a single point, objective-space evaluation must specify how individual candidates are scored,
how attainment is summarized under a fixed candidate budget, and how the full candidate cloud behaves
around the target.

All objective-space computations are performed in a common normalized objective
coordinate system induced by the preprocessing pipeline. The same fitted normalizer is applied to the
training archive, the user targets, and the checked outcomes, so that discrepancy magnitudes are
comparable across objectives and consistent across estimators.

%-------------------------------------------------------------------------------
\subsubsection{Scoring contract in normalized objective space}\label{subsubsec:objective_contract}
%-------------------------------------------------------------------------------

Let $\vect{y}^{\star}\in\mathbb{R}^{m}$ denote the requested objective vector and let
$\widehat{\vect{y}}^{(k)}\in\mathbb{R}^{m}$ denote the checked outcome of the $k$-th candidate, both in
the declared normalized objective space. Candidate mismatch is expressed through the target-centered
residual vector
\begin{equation}
  \vect{r}^{(k)}(\vect{y}^{\star})
  \;=\;
  \widehat{\vect{y}}^{(k)} - \vect{y}^{\star},
  \qquad k=1,\dots,K,
  \label{eq:objective_residual_vector}
\end{equation}
where $m$ is the number of objectives, $K$ is the candidate budget, and $\vect{r}^{(k)}$ measures
signed deviation from the target component-wise. The $j$-th component is
\begin{equation}
  r^{(k)}_{j}(\vect{y}^{\star})
  \;=\;
  \widehat{y}^{(k)}_{j} - y^{\star}_{j},
  \qquad j=1,\dots,m,
  \label{eq:objective_residual_component}
\end{equation}
so $r^{(k)}_{j}>0$ indicates that the candidate is above the target on objective $j$ in the normalized
coordinates, while $r^{(k)}_{j}<0$ indicates that it is below.

A scalar discrepancy score is obtained by applying a distance in this same normalized space. We use
the Euclidean norm as the default scoring rule,
\begin{equation}
  s^{(k)}(\vect{y}^{\star})
  \;=\;
  \big\lVert \vect{r}^{(k)}(\vect{y}^{\star}) \big\rVert_{2}
  \;=\;
  \sqrt{\sum_{j=1}^{m}\Big(r^{(k)}_{j}(\vect{y}^{\star})\Big)^{2}},
  \label{eq:objective_candidate_score}
\end{equation}
where $\lVert\cdot\rVert_2$ denotes the Euclidean norm. The score $s^{(k)}$ supports ranking within a
candidate set for a fixed target: smaller values correspond to checked outcomes that land closer to
$\vect{y}^{\star}$ in the declared objective coordinates.

A small numerical example illustrates the calculation. Consider a bi-objective case ($m=2$) in which
the user target in normalized space is
$\vect{y}^{\star}=(0.40,\,0.70)$ and one checked candidate outcome is
$\widehat{\vect{y}}^{(k)}=(0.45,\,0.62)$. The residual vector is
\begin{equation}
  \vect{r}^{(k)}(\vect{y}^{\star})
  \;=\;
  (0.45-0.40,\; 0.62-0.70)
  \;=\;
  (0.05,\; -0.08),
  \label{eq:objective_example_residual}
\end{equation}
which shows that the candidate overshoots the target by $0.05$ in the first objective and undershoots
by $0.08$ in the second objective. The discrepancy score is then
\begin{equation}
  s^{(k)}(\vect{y}^{\star})
  \;=\;
  \sqrt{0.05^{2}+(-0.08)^{2}}
  \;=\;
  \sqrt{0.0025+0.0064}
  \;=\;
  \sqrt{0.0089}
  \;\approx\;
  0.094,
  \label{eq:objective_example_score}
\end{equation}
and candidates can be ranked by comparing their $s^{(k)}(\vect{y}^{\star})$ values, with the smallest
score ranked as the closest match under the declared contract.

The candidate budget $K$ is part of the evaluation interface. Increasing $K$ can improve attainment by
providing more opportunities to sample a close candidate even if the underlying conditional model is
unchanged. Therefore, all objective-space comparisons fix $K$ within an experiment, and results are
reported separately when $K$ changes.


%-------------------------------------------------------------------------------
\subsubsection{Best-shot attainment and ECDF reporting}\label{subsubsec:objective_ecdf}
%-------------------------------------------------------------------------------

Because inverse samplers return a set, target attainment is summarized per target by the best-shot
score,
\begin{equation}
  s_{\min}(\vect{y}^{\star})
  \;=\;
  \min_{k\in\{1,\dots,K\}} s^{(k)}(\vect{y}^{\star}),
  \label{eq:objective_best_shot}
\end{equation}
which captures the closest checked outcome the model can deliver under budget $K$. Here
$s^{(k)}(\vect{y}^{\star})$ is the per-candidate discrepancy defined by the scoring contract in
Section~\ref{subsubsec:objective_contract}, and $K$ is the number of candidates generated for the
target $\vect{y}^{\star}$. Evaluating a target set
$\mathcal{T}=\{\vect{y}^{\star}_i\}_{i=1}^{N}$ yields the best-shot vector
$\{s_{\min}(\vect{y}^{\star}_i)\}_{i=1}^{N}$, with one value per target.

Reporting only a single aggregate statistic of $\{s_{\min}(\vect{y}^{\star}_i)\}_{i=1}^{N}$, such as a
mean or median, can hide important structure. In particular, two estimators can have similar central
tendency but differ substantially in how often they achieve very small discrepancies or in how large
their failures are on hard targets. To make the full attainment profile visible, we report the
empirical cumulative distribution function (ECDF) of best-shot scores across the target set,
\begin{equation}
  \widehat{F}(t)
  \;=\;
  \frac{1}{N}\sum_{i=1}^{N}\mathbb{I}\!\left[s_{\min}(\vect{y}^{\star}_i)\le t\right],
  \label{eq:objective_ecdf}
\end{equation}
where $N$ is the number of targets in $\mathcal{T}$, $t\ge 0$ is a discrepancy threshold expressed in
the same normalized objective space as $s_{\min}$, and $\mathbb{I}[\cdot]$ is an indicator function
that equals 1 when the condition holds and 0 otherwise. The ECDF therefore maps each threshold $t$ to
a proportion: $\widehat{F}(t)$ is the fraction of targets whose best-shot discrepancy is at most $t$.

The ECDF supports a direct, budget-aware reading of attainment. Fix a discrepancy threshold $t$ on the
$x$-axis. The corresponding $y$-value $\widehat{F}(t)$ reports what proportion of targets are solved
to within that accuracy under budget $K$. Curves that rise faster and lie higher at small $t$ indicate
that an estimator produces close matches more frequently. Conversely, a slow rise and a long right
tail indicate that a subset of targets remains difficult even after taking the best of $K$ samples.

A small numerical example clarifies the calculation. Suppose $N=5$ targets yield best-shot scores
\begin{equation}
  \{s_{\min}(\vect{y}^{\star}_i)\}_{i=1}^{5} = \{0.01,\; 0.04,\; 0.02,\; 0.10,\; 0.03\}.
  \label{eq:ecdf_example_scores}
\end{equation}
Sorting gives $\{0.01, 0.02, 0.03, 0.04, 0.10\}$. For threshold $t=0.03$, four of the five values are
at most $0.03$, so
\begin{equation}
  \widehat{F}(0.03) = \frac{4}{5} = 0.8.
  \label{eq:ecdf_example_value}
\end{equation}
For threshold $t=0.015$, only one value is at most $0.015$, so $\widehat{F}(0.015)=1/5=0.2$. As $t$
increases, $\widehat{F}(t)$ increases in steps until it reaches 1 at $t\ge 0.10$. In the plotted ECDF,
the $x$-axis spans the range of thresholds and the $y$-axis reports the cumulative fraction of targets
whose best-shot discrepancy falls below each threshold, which matches the interpretation used in the
objective-space figures in this chapter.

\subsubsection{Candidate-cloud geometry: bias and dispersion}\label{subsubsec:objective_cloud}
%-------------------------------------------------------------------------------

Best-shot attainment summarizes whether a candidate set contains at least one close match, but it does
not describe how the full set behaves around the target. For decision support, the geometry of the
checked outcome cloud
$\{\widehat{\vect{y}}^{(1)},\dots,\widehat{\vect{y}}^{(K)}\}$ matters because it indicates whether the
inverse sampler produces a concentrated, target-centered set of alternatives or a diffuse or
systematically shifted cloud.

We compute candidate-cloud diagnostics in the same residual space used for scoring. For a fixed target
$\vect{y}^{\star}$, define the within-set mean residual vector
\begin{equation}
  \bar{\vect{r}}(\vect{y}^{\star})
  \;=\;
  \frac{1}{K}\sum_{k=1}^{K}\vect{r}^{(k)}(\vect{y}^{\star}),
  \label{eq:objective_mean_residual_cloud}
\end{equation}
which represents the average offset of the candidate cloud from the target. Candidate-cloud geometry
is summarized using two set-level statistics: systematic offset of the cloud from the target and
within-set dispersion around the cloud mean,
\begin{equation}
  b(\vect{y}^{\star})
  \;=\;
  \frac{\big\lVert \bar{\vect{r}}(\vect{y}^{\star}) \big\rVert_2}{\sqrt{m}},
  \qquad
  v(\vect{y}^{\star})
  \;=\;
  \mathrm{median}_{k}\;
  \frac{\big\lVert \vect{r}^{(k)}(\vect{y}^{\star})-\bar{\vect{r}}(\vect{y}^{\star}) \big\rVert_2}{\sqrt{m}}.
  \label{eq:objective_bias_dispersion}
\end{equation}
Here $b(\vect{y}^{\star})$ reports the magnitude of the mean residual in average per-objective units,
and $v(\vect{y}^{\star})$ reports a robust within-set spread measured as the median distance of
residuals to the cloud mean, again normalized by $\sqrt{m}$ to make values comparable across problems
with different numbers of objectives. The median aggregation in $v$ reduces sensitivity to a small
number of extreme samples and reflects typical within-set variability.

Across a target set $\mathcal{T}$, bias and dispersion yield one value per target, producing vectors
$\{b(\vect{y}^{\star}_i)\}_{i=1}^{N}$ and $\{v(\vect{y}^{\star}_i)\}_{i=1}^{N}$. We report these using
distributional summaries analogous to best-shot attainment. In particular, ECDF curves of $b$ and $v$
support comparisons of how often each estimator produces target-centered clouds and how often it
produces concentrated candidate sets under the declared budget.

%-------------------------------------------------------------------------------
\subsubsection{Ablation protocol and selection criteria}\label{subsubsec:objective_ablation_protocol}
%-------------------------------------------------------------------------------

Inverse decision mapping is sensitive to model capacity. For MDNs, capacity is governed primarily by
the number of mixture components; for CVAEs by latent dimensionality and decoder capacity; and for
INNs by the number of coupling layers and subnet capacity. To avoid attributing performance
differences to under- or over-parameterized settings, we perform an ablation study within each model
family and select one configuration per family for downstream comparison.

The ablation is reported on a representative bi-objective problem derived from COCO BBOB $f_5$
(Linear Slope). For each configuration, we evaluate attainment and candidate-cloud geometry under the
same candidate budget $K$ using the scoring contract defined in
Section~\ref{subsubsec:objective_contract}. Each configuration is evaluated over the same target set
$\mathcal{T}$, producing best-shot scores $\{s_{\min}(\vect{y}^{\star}_i)\}_{i=1}^{N}$ and cloud
statistics $\{b(\vect{y}^{\star}_i)\}_{i=1}^{N}$ and $\{v(\vect{y}^{\star}_i)\}_{i=1}^{N}$.

Hyperparameters are selected using the ECDF profile of best-shot attainment. A configuration is
preferred when its ECDF lies above alternatives at small thresholds, indicating stronger near-target
attainment for a larger fraction of targets. To make comparisons easy to report, we also use
ECDF-derived summaries at fixed thresholds. For two thresholds $t_1<t_2$ chosen in the same normalized
objective space, $\widehat{F}(t_1)$ emphasizes high-precision attainment, while $\widehat{F}(t_2)$
captures broader near-target success. When ECDF curves cross, the selection prioritizes the threshold
regime most relevant to downstream use and is supported by tail-sensitive summaries such as the upper
quantiles of the best-shot distribution.

Candidate-cloud statistics are reported for interpretation rather than used as the primary selection
criterion. In particular, $b$ distinguishes systematic offset from the target, while $v$ distinguishes
diffuse candidate clouds from concentrated sets. These quantities support diagnosing whether a strong
best-shot curve is achieved by consistently target-centered samples or by occasional lucky hits within
a diffuse cloud.

\subsubsection{Hyperparameter ablation and model selection}\label{subsubsec:ablation_decision}

We evaluate hyperparameter variants in decision space using the diagnostics defined in
Section~\ref{sec:eval_design}. For each held-out paired case $(\vect{x}_i,\vect{y}_i)$, we condition on
$\vect{y}_i$, generate a fixed candidate set of size $K$, and compare the resulting sample
distribution against the ground-truth decision $\vect{x}_i$. The ablation is reported on the COCO $f_5$
dataset to match the objective-space analysis and to isolate differences attributable to the inverse
model rather than to changes in the data distribution.

Figure~\ref{fig:f5_pit_ablation} summarizes reliability across hyperparameter variants using PIT
calibration curves. Variants whose curves lie closer to the diagonal exhibit more reliable conditional
uncertainty under repeated conditioning, whereas systematic deviations indicate miscalibration. In
particular, curves that bend away from the diagonal signal that the empirical ranks of the truth do
not match the model’s implied percentiles, meaning that the candidate set does not represent a
trustworthy conditional distribution for downstream use.

Figure~\ref{fig:f5_crps_box_ablation} summarizes probabilistic accuracy across the same variants using
boxplots of per-case CRPS. Lower medians indicate that the sampled conditional distributions place mass
closer to the ground truth without becoming unnecessarily diffuse, while wider boxes and long tails
indicate less stable distributional performance across held-out cases. This view is useful because it
distinguishes variants that occasionally achieve low CRPS from those that do so consistently.

We select the final configuration by reading these two diagnostics jointly: among variants that achieve
a PIT curve close to the diagonal in Figure~\ref{fig:f5_pit_ablation}, we prefer those with lower and
more stable CRPS distributions in Figure~\ref{fig:f5_crps_box_ablation}. Under this criterion, the MDN
variant MDN7 is preferred over other MDN settings because it attains the lowest median CRPS with a
comparatively tight interquartile range in Figure~\ref{fig:f5_crps_box_ablation}, while its PIT curve
in Figure~\ref{fig:f5_pit_ablation} remains closest to the diagonal among the MDN variants. In
practical terms, this combination indicates that MDN7 produces candidate sets that are typically close
to the ground truth (low CRPS) and that the implied uncertainty is comparatively reliable (PIT closer
to uniform). The selected CVAE and INN configurations are chosen analogously by the same joint
calibration--CRPS criterion.

\subsection{Discussion of Results}

Across benchmarks, Gaussian process regression emerges as the most
effective inverse modelling strategy.  Its probabilistic nature allows
uncertainty quantification and smooth interpolation, resulting in low
forward error and generational distance.  Random forests are competitive,
especially when the Pareto front is discontinuous or when training data
is limited, thanks to their ability to capture local patterns and
produce diverse candidates.  Radial basis function networks perform
reasonably well but are sensitive to the choice of centres and require
tuning; they scale poorly with high‑dimensional objectives.  Neural
networks benefit from larger training sets and yield competitive results
once overfitting is controlled.

Data scarcity has a significant impact on inverse model accuracy.
Increasing the number of training samples from 100 to 500 reduces the
RMSE by approximately 40\% across models.  Transfer learning by pooling
  samples from related problems improves performance when tasks share
  similar objective definitions~\cite{Tan2023}.  The plausibility
check effectively warns users when targets lie far from the sampled
Pareto front; in our experiments less than 5\% of queries fell into this
category, highlighting the quality of the data and the models.

Overall, the experiments demonstrate that the proposed framework can
produce high‑quality inverse predictions with modest computational
resources and enables interactive trade‑off exploration.
